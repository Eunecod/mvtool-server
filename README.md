# mvtool-server

Легковесный локальный сервер на базе Python (FastAPI/Uvicorn) для автоматизации процесса обновления `mvtool`. Сервер динамически генерирует JSON-манифест релиза и организует безопасную раздачу файлов обновлений в сети.

## 📡 Эндпоинты и взаимодействие

Сервер предоставляет два основных эндпоинта для работы клиента.

* **Получение манифеста (`GET /update`)**: Возвращает JSON-структуру с информацией о последней версии, ссылкой на релиз и списком доступных ассетов (файлов).
* **Скачивание файлов (`GET /download/{filename}`)**: Безопасно отдает запрашиваемый `.tar` архив из директории `assets`. Передает заголовок `Content-Length` для гарантированной загрузки и стабильной последующей распаковки на стороне клиента.

---

## ⚙️ Описание параметров конфигурации

Программа управляется через файл `lock.json`, который хранит информацию о текущей активной версии релиза на сервере.

### Настройки версии (`lock.json`)
* **`version`**: Номер текущей версии. Сервер автоматически считает это значение, преобразует его в тег формата `v0.0.0` и подставит в ссылки для скачивания при запросе клиента.

Файл версии должен располагаться в одной директории с исполняемым файлом/скриптом. Ниже представлен эталонный пример структуры:
```json
{
  "version": "0.0.0"
}
```

## 📁 Структура сервера

```text
mvtool-server/
├── assets/                        # Директория с готовыми архивами релизов
│   ├── mvtool-x86_64-linux.tar
│   └── mvtool-x86_64-windows.tar
├── lock.json                      # Файл конфигурации текущей версии
└── main.py                        # Исполняемый файл сервера
```

## 🚀 Быстрый старт: Развертывание и настройка окружения

Для запуска сервера в чистом изолированном окружении нужно выполнить следующие шаги.

### 1. Создание виртуального окружения (venv)
Перейдите в терминале в папку проекта и создайте изолированное окружение:

* **Windows:**

  ```bash
  python -m venv .venv
  ```
* **Linux/macOS:**

  ```bash
  python3 -m venv .venv
  ```

### 2. Активация виртуального окружения
* **Windows:**

  ```bash
  ./.venv/Scripts/activate.ps1
  ```
* **Linux/macOS:**

  ```bash
  source .venv/bin/activate
  ```

### 3. Установка зависимостей
* **Windows/Linux/macOS:**

  ```bash
  pip install -r requirements.txt
  ```

### 4. Запуск сервера
* **Windows/Linux/macOS:**

  ```bash
  python main.py
  ```
